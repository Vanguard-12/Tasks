# Structured Output with Pydantic

## Overview

This repository demonstrates how to extract **validated Pydantic objects** from a raw text using LangChain's structured‑output capabilities. Two schemas are supported:

1. **PersonInfo** – information about a person (name, age, profession, skills).
2. **MeetingNotes** – details of a meeting (date, participants, topics, decisions, next steps).

The program automatically decides which schema to use, runs a single‑pass LangChain chain, and prints the resulting Pydantic model via `model_dump()` together with a short human‑readable summary.

## Installation

```bash
# Clone the repository (if you haven't already)
git clone https://github.com/Vanguard-12/Tasks.git
cd Tasks

# Create a virtual environment (Python 3.10+ required)
python -m venv .venv
source .venv/bin/activate   # Windows: .venv\\Scripts\\activate

# Upgrade pip and install dependencies
pip install --upgrade pip
pip install -r requirements.txt
```

If you plan to use the OpenAI provider, copy `.env.example` to `.env` and add your `OPENAI_API_KEY`:

```bash
cp .env.example .env
# edit .env and set OPENAI_API_KEY=your-key-here
```

## Usage

Run the CLI with one of the built‑in examples:

```bash
python cli.py --example person   # extracts a PersonInfo object
python cli.py --example meeting  # extracts a MeetingNotes object
```

Or provide your own text interactively:

```bash
python cli.py
# You will be prompted to paste any text describing a person or a meeting.
```

### Expected output (person example)

```text
{
  "name": "Анна",
  "age": 28,
  "profession": "Python‑разработчик",
  "skills": ["FastAPI", "Docker"]
}
Summary: Person Анна, 28‑year‑old Python‑разработчик with skills FastAPI, Docker.
```

### Expected output (meeting example)

```text
{
  "date": "2024-09-15",
  "participants": ["Иван", "Мария"],
  "topics": ["Бюджет", "Тайм‑лайн"],
  "decisions": ["Утвердить бюджет"],
  "next_steps": ["Отправить отчёт"]
}
Summary: Meeting on 2024-09-15 with participants Иван, Мария – decisions: Утвердить бюджет; next steps: Отправить отчёт.
```

## How it works

1. **Routing** – `router.SchemaRouter` decides whether the input describes a *person* or a *meeting* using a simple keyword heuristic.
2. **Prompt** – `prompt_templates.BASE_PROMPT` contains a placeholder for the format instructions generated by `PydanticOutputParser`.
3. **Chain** – The selected model is passed to `PydanticOutputParser`; the chain is built as `prompt | llm | parser`.
4. **Output** – The parser returns a fully‑validated Pydantic instance, which we dump with `model_dump()` and summarise via `utils.summarize`.
